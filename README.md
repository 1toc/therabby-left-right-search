# Therabby Left-Right Search

A browser-based left-right visual search task prototype for occupational therapy education, observation, and clinical explanation.

This app is designed to help observe how a person searches visual space, especially when there may be differences between the left and right sides of the screen.

## Overview

Therabby Left-Right Search is a simple browser app in which users search for and select blue targets placed across the left and right visual fields.

The task is intended to support observation of:

- Left-right search bias
- Missed targets
- Search order
- First selected position
- Changes after verbal or visual cueing
- Differences between simple and more complex visual conditions

The app is not intended to be a formal assessment tool.  
It is a prototype for clinical explanation, education, and discussion.

## Intended Use

This app may be useful in situations such as:

- Explaining visual search bias to patients, families, or staff
- Demonstrating how missed targets may occur in left or right space
- Supporting occupational therapy education
- Discussing compensatory search strategies
- Exploring how cueing may change visual search behavior
- Connecting screen-based observation to daily activities

Possible clinical themes include:

- Unilateral spatial neglect
- Visual-perceptual difficulties
- Attentional bias
- Visual scanning
- Search strategy training
- Activity analysis in occupational therapy

## Clinical Context

In occupational therapy, visual search difficulties may affect daily activities such as:

- Finding items on a meal tray
- Locating objects on a desk
- Using a smartphone or tablet
- Wheelchair navigation
- Grooming at a sink
- Reading or checking written information
- Avoiding obstacles during mobility

This app is intended to provide a simple way to discuss these issues through a visual and interactive task.

## Important Notice

This app is **not a diagnostic tool**.

It should not be used to determine whether a person has unilateral spatial neglect, visual-perceptual impairment, or any other neurological condition.

Results should be interpreted together with:

- Standardized assessments
- ADL observation
- Clinical reasoning
- Posture and positioning
- Visual function
- Fatigue
- Comprehension
- Motor ability
- Device and input method
- Information from other professionals

This app is a prototype tool for observation, explanation, and education.

## Features

- Browser-based app
- No installation required
- Works on desktop and tablet browsers
- Left and right visual search area
- Blue targets as search targets
- Gray distractors
- Center reference line
- Detection count for left and right targets
- Missed target count
- Time display
- Simple start, finish, and reset controls

## How to Use

1. Open the app in a web browser.
2. Explain the task briefly to the user.
3. Ask the user to find and select the blue targets.
4. Observe the order and pattern of search.
5. Check whether targets are missed on one side.
6. If needed, provide verbal or visual cues.
7. Compare performance before and after cueing.
8. Discuss how the observed search pattern may relate to daily activities.

## Suggested Observation Points

When using this app, observe not only the score but also the process.

For example:

- Where does the user start searching?
- Does the user search both sides equally?
- Are targets missed more often on one side?
- Does the user search from the center outward?
- Are edge targets missed?
- Does cueing improve search?
- Does the user return to previously missed areas?
- Does fatigue change the search pattern?
- Does the user appear confused by the task?
- Does the input method affect performance?
